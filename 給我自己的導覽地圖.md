# Claude Code 原始碼導覽地圖 v2

給不懂程式但有產品直覺的人看的版本。
不綁定任何特定產品——這是你理解「AI Agent 能做什麼」的基礎。

---

## 核心比喻

想像你要開一間公司，這間公司的員工是 AI：

| 角色 | 對應的東西 | 白話 |
|------|-----------|------|
| 🧠 CEO | Claude AI 模型 | 負責思考、決策、判斷下一步 |
| 📋 公司章程 | System Prompt | 告訴 CEO「你是誰、規則是什麼」 |
| 📡 情報室 | Context | 每次開會前收集最新狀況給 CEO |
| 👷 各部門員工 | Tools | 每個人有一個專長，被叫到就去做事 |
| 🔄 工作流程 | Agentic Loop | CEO 下令 → 員工做事 → 回報 → CEO 再決定 |
| 🛡️ 法務部 | Permissions | 某些事情要老闆（你）簽字才能做 |
| 📞 通訊部 | API 層 | 負責跟 Claude 大腦溝通的管道 |
| 🏭 執行部 | Tool Execution | 真正去執行工具、拿結果回來的人 |

---

## 五層架構

```
🧠 智慧層 — AI 知道什麼、能做什麼、怎麼想
🚦 決策層 — 什麼時候執行、是否執行、怎麼執行
🔧 執行層 — 真的去做事、拿到結果
🛡️ 安全層 — 確保人類能控制 AI
📝 溝通層 — 記錄對話、管理狀態
```

---

## 八大核心模組

### 🧠 智慧層

#### 1. Tool 定義 — `src/Tool.ts`
**這是什麼：** 所有工具的「模板」。定義了一個工具該有什麼——名字、描述、需要什麼輸入、回傳什麼結果。

**比喻：** 就像公司的「職位說明書」模板。每個員工（工具）都要照這個模板填寫自己的職責。

**你能做的事：**
- 定義任何你想要的工具（搜尋、分析、寫文章、發送訊息...）
- 指定工具是「唯讀安全的」還是「有風險的」
- 控制工具能不能同時跟其他工具一起執行

---

#### 2. System Prompt — `src/utils/systemPrompt.ts`
**這是什麼：** 把各種指令拼裝成一份完整的「人設卡」給 AI。

**比喻：** 就像新員工入職第一天拿到的「公司手冊」——你是誰、你該怎麼做事、什麼能做什麼不能做。

**你能做的事：**
- 打造任何角色的 AI（行銷助手、客服、研究員、寵物...）
- 設定行為規則和限制
- 動態組裝不同情境的指令

---

#### 3. Context 動態情報 — `src/context.ts`
**這是什麼：** 每次 AI 開始思考前，先收集當前環境的資訊。

**比喻：** CEO 開會前，秘書先準備一份「今日簡報」——現在什麼狀態、有什麼新消息、該注意什麼。

**你能做的事：**
- 讓 AI 知道當前的產品狀態、用戶資料、環境資訊
- 每次對話都自動帶入最新情報
- AI 的回應會更聰明、更貼合情境

---

### 🚦 決策層

#### 4. Agentic Loop — `src/QueryEngine.ts`
**這是什麼：** 整個 agent 的心臟。管理「AI 思考 → 選工具 → 執行 → 看結果 → 再想」的循環。

**比喻：** 公司的工作流程 SOP。CEO 不是回答你一次就下班，而是一直工作直到任務完成。

**你能做的事：**
- 讓 AI 自動串連多個步驟完成複雜任務
- AI 可以根據中間結果改變策略
- 支援長時間、多輪次的對話

---

#### 5. Permission 權限 — `src/utils/permissions/`
**這是什麼：** 決定每個動作要不要先問人類。有多層保護機制。

**比喻：** 法務部。有些事員工可以自己做（查資料），有些事必須老闆簽字（花錢、對外發言）。

**你能做的事：**
- 設計哪些操作 AI 可以自己做，哪些要你確認
- 自動偵測危險操作並攔截
- 追蹤被拒絕的操作，避免 AI 一直重試

**權限的三種模式：**
| 模式 | 意思 |
|------|------|
| Default | 每個有風險的動作都問你 |
| Bypass | 全部自動允許（危險） |
| Auto | 用分類器自動判斷危不危險 |

---

### 🔧 執行層

#### 6. Tools 工具庫 — `src/tools/`
**這是什麼：** 40+ 個現成工具的實作。每個資料夾就是一個工具。

**比喻：** 公司裡每個部門的員工。有人負責搜尋、有人負責寫文件、有人負責跑指令。

**工具分類：**

| 類別 | 包含的工具 | 能力 |
|------|-----------|------|
| 檔案操作 | FileRead、FileEdit、FileWrite、Glob、Grep | 讀寫搜尋檔案 |
| 系統執行 | Bash、PowerShell | 執行系統指令 |
| 網路存取 | WebSearch、WebFetch | 搜尋網頁、抓取內容 |
| AI 組合 | Agent、Skill、ToolSearch | 讓 AI 叫其他 AI 幫忙 |
| 任務管理 | TaskCreate、TaskUpdate、TaskList | 管理背景任務 |
| 互動溝通 | SendMessage、AskUserQuestion | 跟使用者對話 |
| 進階功能 | EnterPlanMode、RemoteTrigger、ScheduleCron | 規劃模式、排程、遠端觸發 |

---

#### 7. API 通訊 — `src/services/api/claude.ts`
**這是什麼：** 負責跟 Claude AI 大腦溝通的管道。發送訊息和工具清單，拿回 AI 的決策。

**比喻：** 通訊部。把會議記錄和可用資源清單送給 CEO，CEO 回覆他的決定。

**你能做的事：**
- 把你定義的工具清單傳給 AI，讓它自己選要用哪個
- 控制 token 使用量（省錢）
- 處理 API 錯誤和重試

---

#### 8. Tool Execution 工具執行 — `src/services/tools/toolExecution.ts`
**這是什麼：** 真正去執行工具的流程。驗證 → 檢查權限 → 執行 → 處理結果 → 回傳給 AI。

**比喻：** 執行部門。收到 CEO 的指令後，先確認合不合規，然後派人去做，做完回報結果。

**執行流程：**
```
1. 驗證輸入 → 參數對不對？
2. 檢查權限 → 可以做嗎？
3. 執行前 Hook → 有沒有前置作業？
4. 執行工具 → 真的去做
5. 處理結果 → 整理回傳資料
6. 執行後 Hook → 有沒有後續處理？
7. 回傳給 AI → CEO 看結果
```

## 進階能力（你未來可能用到的）

| 能力 | 在哪裡 | 什麼意思 |
|------|--------|---------|
| Agent 套娃 | `src/tools/AgentTool/` | AI 可以叫另一個 AI 去做子任務 |
| Skill 技能包 | `src/skills/` | 把常用的 prompt + 流程包裝成可重複使用的技能 |
| MCP 協議 | `src/services/mcp/` | 讓 agent 連接外部工具和資料源的標準協議 |
| 對話壓縮 | `src/services/compact/` | 對話太長時自動摘要，省 token |
| 背景任務 | `src/tasks/` | 讓工具在背景跑，不卡住主對話 |
| Hook 擴展 | `src/hooks/` | 在工具執行的各個階段插入自訂邏輯 |
| 排程觸發 | `src/tools/ScheduleCronTool/` | 讓 agent 按照時間表自動執行 |
| 多 Agent 協作 | `src/coordinator/` | 多個 agent 一起合作完成大任務 |

## 一句話總結

> Claude Code 的原始碼告訴你：一個 AI Agent 就是**一個有人設的 AI**，配備**一組工具**，透過**循環思考**來完成任務，在**關鍵時刻問人類確認**，而且能**串接任何外部服務**。你可以用這套 pattern 打造任何 AI 產品。
